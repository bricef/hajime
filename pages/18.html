Lisp has macros

It's a bit strange to introduce the topic so early, but this is just about planting the seed that will grow into something else.

Macros are read-time code transformations.

Macros take in a list representing the code and return a list representing the transformed code.

What macros are actually useful for is to add new constructs to the language.

Unlike in other languages, if a feature is missing, you have the power to add it.

Let's imagine we want to write an 'unless' (this is a typical macro example)

We could try writing

; this will fail (defn unless [condition form] (if expr nil form))

Why don't you try it to see the problem

(unless false (prn "should print")) (unless true (prn "should not print"))

Why is this going wrong?
